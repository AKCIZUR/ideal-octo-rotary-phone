# VCCSD Docs

Stabilní základ pro dokumentaci, který lze rychle upravit pro konkrétní projekt a dlouhodobě používat bez složité údržby.

## Co šablona řeší

- Markdown-first obsah
- přehlednou navigaci
- tmavý vizuální styl
- automatické nasazení přes GitHub Actions
- snadné verzování dokumentace
- čistý základ bez zbytečné režie

## Jak s ní pracovat

1. uprav názvy a odkazy v `astro.config.mjs`
2. vlož vlastní obsah do `src/content/docs/`
3. nastav `SITE_URL` a `BASE_PATH` ve workflow
4. pushni změny do GitHubu
5. GitHub Pages se nasadí automaticky

## Kde začít

Podívej se nejdřív na:

## Výhody

- rychlý start
- minimum závislostí
- dobrá čitelnost
- vhodné pro interní i veřejné docs
- snadno rozšiřitelný sidebar
- připravené místo pro průvodce, FAQ a changelog

## Doporučený pracovní postup

```bash
npm install
npm run dev
```

Pak už stačí jen přidávat další soubory do `src/content/docs/`.

## Co je na tom důležité

Cílem není vytvořit jen hezkou šablonu. Cílem je mít dokumentační základ, který vydrží dlouho, dobře se udržuje a není svázaný s těžkopádným systémem.
